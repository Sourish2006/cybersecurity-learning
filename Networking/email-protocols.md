# Email Protocols (SMTP, POP3 and IMAP)

## 📚 Introduction

Email protocols define how emails are sent, received and stored over networks.

The three most common email protocols are:

- SMTP
- POP3
- IMAP

---

## 📤 SMTP (Simple Mail Transfer Protocol)

SMTP is responsible for sending emails between clients and mail servers.

Default Ports:

- 25
- 587
- 465 (Secure SMTP)

Examples:

- Gmail sending messages
- Outlook sending emails

---

## 📥 POP3 (Post Office Protocol Version 3)

POP3 retrieves emails from a mail server and usually downloads them to a local device.

Default Ports:

- 110
- 995 (Secure POP3)

Characteristics:

- Downloads emails locally
- May remove emails from the server
- Suitable for single-device usage

---

## 📬 IMAP (Internet Message Access Protocol)

IMAP allows users to access and synchronize emails across multiple devices.

Default Ports:

- 143
- 993 (Secure IMAP)

Characteristics:

- Emails remain on the server
- Supports synchronization
- Ideal for multiple devices

---

## ⚖ POP3 vs IMAP

| Feature | POP3 | IMAP |
|----------|------|------|
| Email Storage | Local Device | Mail Server |
| Synchronization | No | Yes |
| Multi-device Support | Limited | Excellent |
| Accessibility | Single Device | Multiple Devices |

---

## 🌍 Real-World Applications

Email protocols are used in:

- Gmail
- Outlook
- Yahoo Mail
- Corporate Email Systems

---

## 🔑 Key Takeaways

- SMTP is used for sending emails.
- POP3 retrieves emails to a local device.
- IMAP synchronizes emails across multiple devices.
- Secure versions use encrypted ports.
- Email protocols are essential for Internet communication.
