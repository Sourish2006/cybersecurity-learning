# 🌐 HTTP & HTTPS

## 📘 What is HTTP?
HTTP is a protocol used to transfer data between client and server.

## 🔐 HTTPS:
- Secure version of HTTP
- Uses encryption (SSL/TLS)

## 🔑 Difference:
- HTTP → Not secure
- HTTPS → Secure

## 🧠 My Understanding:
HTTP is used for communication, but HTTPS ensures data is encrypted and secure.

## 🔄 Client-Server Communication

- Client → Browser (user side)
- Server → Website server

👉 Flow:
Client → Request → Server  
Server → Response → Client

  ## 📩 HTTP Request

A request is sent by the client to the server.

### 🔑 Components:
- Method → GET, POST
- URL → /index.html
- Headers → extra info (browser, type)
- Body → data (in POST request)

---

## 📤 HTTP Response

A response is sent by the server to the client.

### 🔑 Components:
- Status Code → 200, 404, 500
- Headers → content type
- Body → actual data (HTML, JSON)

---

## 📊 Common Status Codes

- 200 → OK (Success)
- 404 → Not Found
- 500 → Server Error

---

## 🔐 HTTP vs HTTPS

- HTTP → Not secure (data visible)
- HTTPS → Secure (encrypted using SSL/TLS)


## 🧠 My Understanding:
HTTP works on a request-response model where the client asks for data and the server responds with it.HTTPS adds security by encrypting the communication.
